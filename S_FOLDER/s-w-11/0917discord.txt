인덱스. 쿠키와 세션이 무엇인지, 왜 나왔는지.
쿠키는 서버가 사용자의 웹 브라우저에 전송하는 작은 데이터 조각이다. 이 데이터 조각을 사용자와 주고 받으며 세션 관리, 개인화, 트래킹 등을 하게 된다.
세션은 사이트와 특정 브라우저 사이의 상태를 유지하는 것이다. 세션 아이디를 발급하고 세션 아이디를 쿠키에 저장한다. 서버는 세션 아이디를 확인해 알맞은 로직을 처리하게 된다.
쿠키와 세션 같은 경우, http의 특징인 비연결지향, 무상태라는 특성으로 인해 현재 사용자의 상태가 로그인 상태인지, 로그아웃 상태인지 등을 모른다. 그래서 현재 사용자 상태를 세션에 저장해서 쿠키에 담아 서버로 보내어 통신 할 때 사용자의 상태를 확인 할 수 있도록 하기 위해 사용된다.

유저는 빌트인 모델과 커스텀 유저모델을 쓸 때 뭐가 다른지. 달라지는 이유.
유저 빌트인 모델 같은 경우, 수정, 추가 등의 작업을 할 때 작업해야 하기 굉장히 번거로워지며, User 모델이 바뀔 때도 여러 추가 작업을 해야한다. 그렇기에 커스텀 유저 모델을 사용하게 된다.
유저 정보를 건드리는 유저 크리에이션 폼과 유저 체인지 폼 두가지만 커스텀 유저 모델 업데이트 해야한다.
