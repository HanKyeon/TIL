
# 개발

Date 객체를 주로 다뤘고, 여러 라이브러리를 확인했음.

상당 수 path의 사용법과 API 요청의 사용법이 달랐음.

Mobx를 사용하는 방식을 배웠음. Store, Repo, Model을 나눠 관리했음.

특히 getter setter를 통해 데이터를 숨기고 API 요청 역시 어떻게 잘 숨기는 것을 보고 놀랐음.

CSS 방법론인 BEM을 생각보다 다들 잘 쓰고 계신걸 보고 놀랐음.

의존성으로 인해 버전이 맞지 않을 때, 강제로 맞추는 resolution 옵션에 대해 알았음.
- 그걸 알아가는 와중에 부수적으로 yarn why {{패키지 모듈}} 하면 어디어디에 의존성이 있는지 띄워주는 것을 알았음.
- 디펜던시 트리 역시 보기 좋게 만들 수 있겠구나 느꼈음!
- npm이든 yarn이든 링크로 직접 받을 수 있다.

드로어, 셀렉터, 상태 연결 등의 문제를 해결해 개발했다.

모바일 설정창을 맡았고, 정상 구동되는 것을 확인했음.

Mobx의 경우, Loading, error 등을 ReactQuery처럼 알려주지 않고, 직접 알려줘야 한다.
- 상당히 난해한데, 그냥 상태 하나 더 두고 관리하면 될 것 같다.
- Mobx로 직접 구현하는 것을 보다보니 왜 ReactQuery가 hook과 provider로 관리하는지 알 것 같음.
