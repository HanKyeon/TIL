- tcp udp 차이 재정리
  - tcp : Transmission Control Protocol의 약자로, TCP는 일반적으로 IP를 함께 사용한다.
  - IP가 데이터의 배달을 처리, TCP는 패킷을 추적하고 관리한다.
  - 특징으로는 연결지향 방시그로 패킷 을 교환하며, 3way handshaking을 통해 연결하고 4way를 통해 해제한다는 것, 흐름 제어 및 혼잡 제어가 좋고 신뢰성을 보장하며, UDP보다 느리고 Full Duplex Point to Point 방식을 이용한다는 것이다.
  - 일반적으로 서버 소켓은 연결만을 담당하고, 연결 과정에서 반환된 클라이언트 소켓은 데이터의 송수신에 사용된다. 서버와 클라는 1대1로 연결이 되며, 스트림 전송이기에 전송 데이터 크기에 제한이 없다. 패킷에 대한 응답을 해야하기에 성능이 좋지 않으며, streaming 서비스에 불리하다.
  
  - udp : User Data Protocol의 약자로, 비연결형 프로토콜이다. 즉, 연결을 위해 할당되는 논리적인 경로가 없다.
  - 그렇기에 각 패킷은 다른 경로로 전송되고, 각각의 패킷은 독립적인 관계를 지니게 된다. 데이터를 서로 다른 경로로 독립적으로 처리하게 하는 프로토콜.
  - 특징으로는 비연결형 서비스로 데이터그램 방식을 제공한다는 점, 정보를 주고 받을 때, 보내거나 받는다는 신호 절차가 없다는 점, 헤더의 CheckSum 필드를 통해 최소한의 오류만 검출하기에 신뢰성이 낮으며, TCP/IP보다 빠르다는 점이 특징이다.
  - 신뢰성보다 연속성이 중요한 실시간 스트리밍에 자주 사용된다.
  - 연결 자체가 없어서 connect가 업으며 서버 소켓과 클라 소켓의 구분이 없다.
  - 소켓 대신 IP를 기반으로 데이터를 전송한다.
  - 서버와 클라는 1대1 1대n n대m 등으로 연결될 수 있다.
  - 메시지 단위로 전송되며 그 크기는 65535바이트로 제한되어 크기가 초과하면 잘라서 보낸다.
  - 흐름제어가 없어서 패킷의 오류를 알 수 없다. 파일 전송 등 신뢰성이 필요한 서비스보다 성능이 중요한 경우 사용된다.
