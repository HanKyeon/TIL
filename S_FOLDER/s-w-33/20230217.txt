Keep
- 자발적으로 프로젝트를 하려는 것이 좋았다.
- git conflict가 굉장히 적었던 점이 만족스러웠다.
- git commit message는 좋은 것 같다. 누가 무슨 작업을 했을 때 코드 조정이 편했다.

Try
- 타입 스크립트를 사용했더라면 시간이 들어도 편해지는 부분이 있었을 것 같다.
- 이번에는 내가 일반 css를 사용하자고 했지만 다음에는 모듈형 css를 하고 싶다.
- 지라 이슈에 대한 리뷰를 진행 했다는 예시가 있었는데, 해당 부분을 시행하면 더 좋을 것 같다.

Problem
- 상태 관리가 꼬였던 것이 많았기에 상태관리 부분의 기획을 더 철저히 했어야 했다.
- 반복되는 로직의 작성을 hook으로 줄였다면 개발 시간이 더 남았을 것 같다. 특히나 모듈형 css를 쓰지 않는 우리 팀의 경우는 더더욱.
- 학습한 것을 공유하는 부분에서 약했던 것 같다.
- 리팩토링 할 코드들이 많다 느꼈다. 처음부터 리팩토링이 최소한 필요하도록 하는 것이 좋다.
