#### 삽희는 keep 과 problem을 정리한다.

Keep 예시
- 지라 스프린트 종료 후 피드백 한 것
- 컨벤션 상세화 및 가돇어 증가
- 하드 코딩 대신 로직을 모듈화해서 재사용성 높인 것.
- 지라를 활용한 버그 리포트
- 상호 간 코드 리뷰
- 트러블 슈팅 노션에 상세 정리.

Problem 예시
- 시간 부족으로 자기 영역 외에 모르고 지나친 것.
- 기능 구현에 매여 문제 해결 능력이 부족한 경우.
- 트러블 슈팅 후, 기록 없이 넘어가서 같은 이슈가 발생 했을 때 바로 해결하지 못한 것. * 이거 중요한듯. 에러별로 적어두자.
- 커밋 메시지를 컨벤션 맞춰서 작성.
- 하드코딩 프로그램.
- 배포 중 was 다운으로 인해 테스트가 불가 했던 것.

Try 예시
- 깃에 리드미나 위키에 플젝 진행상황 상세하게 기록하고 공유
- 커밋 메시지 직관적 작성
- 그날 그날의 트러블 슈팅 기록
- 무중단 배포 시스템 구축
- 이슈 세분화, 할당, 공유.
- 기능 구현에서 어려운 부분이 있다면 SOS 요청 빠르게 하기.
