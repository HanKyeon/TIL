## 평가 방법
- 기준지어서 선별하고 심사위원의 심사를 통해 반 별 우수팀을 만든다.
- 본인들이 만든 서비스를 발표하며 프로젝트 목표를 좀 더 명확히 한다.
- 발표회는 캠퍼스 분리하고 전체 1위 뽑는다.
- 자율 때는 본선 결선 발표회 한다.
- UCC 경진대회도 있다. 프로젝트 종료 시 제출해야 한다. 서비스를 뽐내면 된다고 한다. 플젝 진행과정과 결과물을 영상으로 포폴화.
- ucc 경진대회는 직접 골라서 한다. 기획의도, 완성도, 컨텐츠 설명 등. 서비스 설명, 시연이 잘 녹아야 한다.
- 싸피 미디어랩 업그레이드 기회. 업그레이드를 통해 추가 촬영 영상소스 추가 영상 효과 추가 등 받는다. 라이선스를 받는다.
- 우수 플젝 시상. 프로젝트 아이디어, 주제 창의성, 프로그래밍 스킬, 서비스 어필하는 발표력. 공/특은 각 반 별 우수팀 선별. 최종 결선 이후 최우수 우수 선별.

### 멀티 스레드 채팅 서버 구현
유저, 최대 5명까지 텍스트기반 채팅을 만들어봐라.
- 프로세스와 스레드에 관한 기초 지식을 쌓는다.
- 네트워크에 대해서도 주요 내용을 정리하고 기초 지식을 쌓는다.
- 멀티스레드 채팅 서버, 클라이언트를 직접 구현하면서 스레드, 네트워크 관련 이해를 조금 더 구체화 한다.
- 프로세스와 스레드의 개념, 네트워크 통신&I/O 실 구현을 통한 감각 생성 등을 얻어갈 수 있다.
- 자바 프로세스 Thread dump 수집/분석기 개발.
##### 프로세스
- 운영체제가 관리하는 하나의 실행 단위.
- 보통 GUI 환경에서 아이콘 마우스 더블 클릭 시 해당 프로그램을 운영체제의 메커니즘에 기반한 단위로 메모리에 상주시킴.
- 각 운영체제별 프로세스 조회 및 관리 방법 제공됨. 윈도우 기반 서버가 아닌 우분투 os로 서버를 구현한다. 그러면서 데이터를 공부하면 좋다.
##### 스레드
- 프로세스 내에서 실행되는 작업 흐름의 단위
- CPU는 한 번에 하나의 명령 수행을 하게 되는데, 스레드를 활용하여 동일 프로세스 내 여러 독립된 작업을 동시에 수행하는 듯한 효과를 볼 수 있음
- 스레드 예시. 동영상 플레이어. 하드웨어 스레드, 소프트웨어 스레드. 한 번에 싫애하는 UX를 제공해야 한다.
- 스레드도 준비 실행 대기 반복하다 종료 되는, Priority 근거 상태 전이. 운영 체제에서 어떻게 우선순위를 처리하는지 등을 처리하고 배울 수 있으면 좋을 것이다.
- 어플부터 통신 단말 장치까지 모든 과정을 알면 좋다. 
- 물리계층L1의 bit 신호 송수신 및 MAC Adress 등.
- 데이터 링크 계층L2의 물리적 신호 오류감지, 수정 및 전송, 이더넷이 대표적이다. Ethernet
- 네트워크계층L3 Ip계층. 패킷 전송 경로 설정, 라우터의 라우팅 테이블을 근거로 경로 결정, 인터넷 프로토콜이 대표적.
- 전송 계층L4 TCP. 전송 제어. 오류검출, 재전송 등. TCP, UDP가 대표적
- 세션 계층L5. 응용 프로세스에서의 통신 관리 및 유지. 포트 바인딩
- 표현 계층L6. 데이터 압축, 암호화.
- 응용 계증L7. End application. HTTP, FTP 등. HTTP 하나는 정리하면 좋다.

#### 소켓 통신
- 소프트웨어로 작성된 추상적인 개념의 통신 접속점.
- 운영체제 레벨에서 제공되어진다고 볼 수 있음.
- 선택한 언어에서 소켓 통신을 어떻게 구현 할 수 있는가? 찾아보면 된다. 

- 파이썬의 경우는 import socket 등 해야한다. 공식 API 문서를 참고하거나 도서 혹은 블로그 검색을 해보면 소켓 예제를 찾을 수 있다.
- 검색 엔진을 돌려보고, 나만의 방식으로 이해하고 활용해보는 것은 필요하다.
- 스레드 및 소켓을 경험 해보는 것은 자바나 파이썬 선택하기를 추천한다.

#### 스레드 프로그래밍
- 클라이언트 프로세스는 서버로부터 소켓 통신을 통해 메시지를 읽어들이고 콘솔에 출력하는 작업과, 콘솔창의 키보드 입력 텍스트를 소켓 통신을 통해 서버로 전송하는 작업
- 채팅 서버 역시 최대 5명까지의 소켓 세션이 유지되고 있는 클라이언트들로부터의 메시지 수신과 송신을 동시에 수행해야 한다.
- 메시지 수신/송신 두가지 기능을 동시에 처리 할 수 있어야 하며, 이 때 필요한 것이 각각의 작업을 동시에 별도 처리 할 수 있는 스레드 프로그래밍이 되어야 한다.

- 동영상은 화면 / 소리 양쪽.
- 채팅은 키보드 입력이 소켓 세션을 통해 데이터를 파악해야 하고, 쓰는 것과 듣는 것에서 쓰레드가 필요하다.
- 소켓 예제를 찾아보면 굉장히 잘 나온다.
- 소켓과 마찬가지로 선택 언어에서 어떻게 스레드를 활용 할 수 있는지와 같은 기본적인 사용법을 찾아보는 건 개발자의 기본 덕목이다.
- 공식 API 보면 쓰레드 사용하는 방법이 나온다. threading.
- 스레드에 대한 감각, 네트워크에 대한 감각을 얻어야 한다. 하루에 끝내고 계속 하는 것이다.
